# Завдання 1. Вивчити та зрозуміти переваги та недоліки інструментів.

## Інструменти, розглянуті на уроках, включають як засоби деплою (Nginx, Heroku, AWS), так і вбудовані засоби безпеки Django.

| Інструмент / Метод        | Тип                   | Переваги                                                                                  | Недоліки / Нюанси |
|---------------------------|------------------------|-------------------------------------------------------------------------------------------|--------------------|
| **Шаблони Django**        | Вбудований захист      | Автоматично екранують HTML, захищають від більшості XSS-атак                              | Не захищають у специфічних HTML-атрибутах (`<style>`, `onclick=`) |
| **Django ORM**            | Вбудований захист      | Екранує SQL-запити, захищаючи від SQL injection                                           | При використанні `extra()`, сирого SQL або RawSQL потрібно вручну екранувати |
| **Вбудований CSRF-захист**| Вбудований захист      | Захист від CSRF-атак на основі перевірки токенів                                          | Може бути вимкнено через `csrf_exempt`, що небезпечно |
| **SSL/HTTPS**             | Захист з’єднання       | Шифрує дані, запобігає перехопленню персональної інформації                               | Потрібна конфігурація Nginx та налаштування параметрів Django |
| **Nginx**                 | Вебсервер / Проксі     | Висока продуктивність, статика, балансування, SSL, проксі                                 | Вимагає ручного налаштування |
| **Heroku (PaaS)**         | Хостинг                | Простий деплой, не потрібно керувати сервером                                             | Безкоштовний тариф "спить", немає постійного сховища |
| **AWS EC2 (IaaS)**        | Хостинг                | Гнучкість, масштабованість, контроль на рівні ОС                                          | Вимагає досвіду адміністрування |
