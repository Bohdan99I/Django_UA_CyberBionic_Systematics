# Порівняння інструментів

## Завдання 1, 4, 6 – Порівняння інструментів деплою

На уроці розглядалися такі інструменти деплою Python/Django-застосунків:

| Інструмент          | Переваги                                                                                          | Недоліки                                                                                  |
|---------------------|---------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------|
| **Gunicorn**        | Простий у використанні, швидкий, добре інтегрується з Django, підтримує кілька workers           | Не є повноцінним веб-сервером, не обслуговує статичні файли, потрібен Nginx як фронтенд     |
| **uWSGI**           | Дуже висока продуктивність, величезна кількість налаштувань, підтримує багато протоколів          | Складна конфігурація, важка документація                                                  |
| **Nginx**           | Швидкий, відмінно віддає статику та медіа, proxy, load balancing, SSL, кешування                 | Потребує ручного редагування конфігураційних файлів                                        |
| **PythonAnywhere**  | Безкоштовний базовий план, деплой у один клік, вбудована БД і консоль, не потрібен VPS            | Обмеження CPU/пам’яті на безкоштовному тарифі, немає root-доступу, обмежений контроль     |
| **Docker**          | Повна ізоляція середовища, однаковий результат на всіх машинах, легко масштабувати               | Складніше налаштування для новачків, більші образи, потребує знання Docker                  |
| **systemd + venv**  | Повний контроль над сервером, автозапуск після перезавантаження, максимальна гнучкість           | Потрібен власний VPS/сервер, ручне налаштування безпеки, оновлень, firewall тощо           |
