# Це практичне завдання вимагає виконати такі кроки:


1. Вибір середовища: Обрати віддалений Linux-сервер.

2. Вибір інструментів: Використати Nginx та uWSGI для деплою.

3. Кроки деплою:
    - Підготовка проєкту: Змінити налаштування проєкту для продакшну (наприклад, DEBUG = False , коректне налаштування SECRET_KEY ).
    - Встановлення uWSGI: pip install uwsgi.
    - Налаштування uWSGI: Створення конфігураційного файлу uWSGI.
    - Налаштування Nginx: Налаштування проксіювання запитів до uWSGI та обслуговування статичних файлів.
    - Запуск: Запуск uWSGI та Nginx.

---

## Типові проблеми та помилки при деплої:

1. Помилка 502 Bad Gateway / Connection Refused: Часто означає, що Nginx не може підключитися до uWSGI-процесу (uWSGI не запущений, запущений на неправильному сокеті/порті або файрвол блокує з'єднання).

2. Помилка 404 для статичних файлів: Nginx не налаштований належним чином для обслуговування статичних файлів (потрібно перевірити директиву location та шлях root/alias).

3. Internal Server Error (500) від Django: Зазвичай через помилки в налаштуваннях Django:
    - DEBUG = True у продакшні (потрібно змінити на False).
    - Неправильно налаштований ALLOWED_HOSTS (додайте доменне ім'я/IP-адресу сервера).
    - Неправильне налаштування SECRET_KEY (має бути у змінній середовища або файлі з обмеженим доступом).
