# Nginx

## На основі розглянутих матеріалів, Nginx використовується як:


1.  Проксирувальний сервер: Приймає запити від клієнтів і передає їх далі, згідно з інструкціями в конфігураційному файлі, зокрема до uWSGI-процесу, який запускає Django-застосунок.


2.  Сервер для обслуговування статичних файлів: Хоча це не зазначено явно як окремий параметр, це ключова роль Nginx у деплої Django (розвантажує Django від роботи зі статикою).

---

## Основні аспекти, які потрібно вивчити в документації:

1.  Конфігураційні файли: Як структурується конфігурація Nginx (директиви server, location, http).

2.  Директива location: Використовується для визначення, як Nginx обробляє запити для певних URL-шляхів. Це критично для:

    - Проксіювання до uWSGI: Налаштування proxy_pass або uwsgi_pass для динамічних запитів.

    - Обслуговування статичних файлів: Налаштування alias або root для шляхів, що ведуть до статичних файлів (як static/ або media/).


3.  Перевірка стану: Перевірка, чи Nginx встановлено та запущено (перейшовши на yourserver.com:80 і побачивши повідомлення «Welcome to nginx!»).
